<objective>
Add a segment-level error boundary at `src/app/error.tsx` so that render crashes in non-gameroom pages (login, profile, collections, etc.) show a minimal fallback UI instead of a white screen, and report the error to Sentry.

Purpose: `global-error.tsx` (Phase 10) only fires when the root layout itself crashes. Page-level crashes in non-gameroom routes are currently uncaught — a render error in `/login` or `/profile` produces a white screen with no Sentry event. This plan closes that gap (OBS-03).

Output: One new file — `src/app/error.tsx`.
</objective>
