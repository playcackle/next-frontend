<objective>
Create a custom React class error boundary (`GameroomErrorBoundary`) that silently attempts one recovery before showing a minimal fallback UI, then mount it in `src/app/gameroom/layout.tsx` to wrap all gameroom pages.

Purpose: `error.tsx` (Next.js convention) renders the fallback immediately on any crash — silent-first-attempt recovery is architecturally impossible with that file. A custom class component with a two-state machine (`hasError` + `recoveryAttempted`) is required. On the first crash it renders nothing and schedules `setState({ hasError: false, recoveryAttempted: true })` via `setTimeout(fn, 0)`, triggering a re-render attempt. If the re-render crashes again, the fallback renders. This satisfies OBS-04.

Output: Two files modified/created — `GameroomErrorBoundary.tsx` (new), `layout.tsx` (modified to wrap children).
</objective>

GameroomErrorBoundary state machine (required design):
```
State: { hasError: boolean, recoveryAttempted: boolean }

Initial: { hasError: false, recoveryAttempted: false }

On crash (getDerivedStateFromError): { hasError: true }
  -> render(): return null  (silent — recovery in flight)
  -> componentDidCatch(): if !recoveryAttempted -> setTimeout(() => setState({ hasError: false, recoveryAttempted: true }), 0)

On second crash (if first recovery failed):
  State: { hasError: true, recoveryAttempted: true }
  -> render(): return <MinimalFallback />  (unrecoverable)

On successful recovery:
  State: { hasError: false, recoveryAttempted: true }
  -> render(): return this.props.children  (back to normal)
```

4. Interface / state types:
```typescript
interface Props {
  children: React.ReactNode;
}

interface State {
  hasError: boolean;
  recoveryAttempted: boolean;
}
```

5. Class: `export class GameroomErrorBoundary extends React.Component<Props, State>`

6. Constructor: initialize state as `{ hasError: false, recoveryAttempted: false }`

7. `static getDerivedStateFromError(_error: Error): Partial<State>`:
   - Returns `{ hasError: true }` only
   - Does NOT call captureException here (no side effects allowed in getDerivedStateFromError)

8. `componentDidCatch(error: Error, _info: React.ErrorInfo)`:
   - Call `captureException(error, { tags: { boundary: "gameroom" } })`
   - Do NOT pass `info.componentStack` as a tag value — it is multi-line and would be truncated by Sentry (document in comment)
   - Gate the auto-reset: `if (!this.state.recoveryAttempted)` then schedule `setTimeout(() => this.setState({ hasError: false, recoveryAttempted: true }), 0)`
   - Without the gate, a persistent error causes an infinite setState->crash loop

9. `render()` — three cases in order:
   a. `this.state.hasError && this.state.recoveryAttempted` -> return minimal fallback (unrecoverable)
   b. `this.state.hasError && !this.state.recoveryAttempted` -> return `null` (silent recovery in progress)
   c. Default -> return `this.props.children`

10. Minimal fallback UI (case a):
    - Match visual style: `padding: "2rem"`, `color: "white"`, `background: "#0a0a1f"`
    - User-safe copy only: "Game connection lost" heading, "Return to lobby and rejoin." paragraph
    - NO `error.message`, NO `error.stack`, NO stack trace, NO internal React state in the rendered output

Do NOT:
- Import from "@sentry/nextjs" directly
- Pass `componentStack` as a Sentry tag value
- Omit the `recoveryAttempted` gate (causes infinite loop on persistent errors)
- Create src/app/gameroom/error.tsx (would shadow or conflict with this boundary)
