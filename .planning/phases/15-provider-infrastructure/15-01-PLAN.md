<objective>
Write the SQL migration that hardens the handle_new_user() PostgreSQL trigger to work correctly for Google and Discord OAuth sign-ins without producing NULL constraint errors.

Purpose: The existing trigger reads `raw_user_meta_data ->> 'name'` which is set during email/password signup but absent during OAuth sign-in. Google uses `name`, Discord uses `user_name` (Supabase-normalized from Discord's `username`). Without a COALESCE fallback chain the first OAuth sign-in will fail with a NULL constraint error on the players table.

Output: A single SQL migration file ready to be run in the Supabase SQL Editor.
</objective>
