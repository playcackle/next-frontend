  <what-to-do>
    Complete all three setup actions below. They can be done in any order, but the migration must be applied before any OAuth sign-in test.

    ---

    **A. Apply the SQL migration (SETUP-04)**

    1. Open Supabase Dashboard → SQL Editor
    2. Before running: go to Table Editor → public.players and note the exact column names and which are NOT NULL. If the schema differs from the migration's assumed columns (id, name, email, avatar_url), edit the migration file first.
    3. Copy the contents of `supabase/migrations/20260319000000_handle_new_user_oauth.sql` and paste into the SQL Editor
    4. Run the query
    5. Confirm: "Success. No rows returned" (CREATE OR REPLACE FUNCTION produces no rows)
    6. Verify the function exists: run `SELECT proname FROM pg_proc WHERE proname = 'handle_new_user';` — should return one row

    ---

    **B. Register Google OAuth app (SETUP-01)**

    1. Go to https://console.cloud.google.com → APIs & Services → Credentials
    2. Create credentials → OAuth 2.0 Client ID → Application type: Web application
    3. Name: "Cackle" (or similar)
    4. Authorized JavaScript origins: Add your app's base URL (e.g. `https://your-app.com` or `http://localhost:3000` for dev)
    5. Authorized redirect URIs: Add exactly `https://<your-supabase-project-ref>.supabase.co/auth/v1/callback`
       - Copy this URL from Supabase Dashboard → Authentication → Providers → Google (shown before you enable it)
       - Do NOT type this manually — copy-paste to avoid mismatch
    6. Save → Download or copy the Client ID and Client Secret

    ---

    **C. Register Discord OAuth app (SETUP-02)**

    1. Go to https://discord.com/developers/applications → New Application
    2. Name: "Cackle" (or similar) → Create
    3. Go to OAuth2 tab
    4. Under Redirects → Add Redirect: paste exactly `https://<your-supabase-project-ref>.supabase.co/auth/v1/callback`
       - Copy from Supabase Dashboard → Authentication → Providers → Discord
    5. Copy the Client ID and Client Secret (Reset Secret if needed)

  </what-to-do>
  <resume-signal>Type "migration applied, apps registered" when all three actions are complete</resume-signal>
