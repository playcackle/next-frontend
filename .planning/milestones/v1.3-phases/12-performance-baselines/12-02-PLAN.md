  <name>Task 2: Instrument useGameEvents with performance.now() timing probes</name>
  <files>src/app/gameroom/hooks/useGameEvents.ts</files>
  <action>
    Add `performance.now()` timing around event handler dispatches in `useGameEvents.ts`.

    The goal is to measure:
    1. Handler execution time per event type (in ms)
    2. Relative frequency of lobby_tick vs other events

    Wrap each `onEvent` registration to add timing in dev mode only. Replace:
    ```ts
    onEvent("lobby_tick", (data: LobbyTickPayload) => handleLobbyTickRef.current(data)),
    ```

    With:
    ```ts
    onEvent("lobby_tick", (data: LobbyTickPayload) => {
      const _t0 = performance.now();
      handleLobbyTickRef.current(data);
      if (process.env.NODE_ENV === 'development') {
        console.log(`[PerfProbe] lobby_tick: ${(performance.now() - _t0).toFixed(3)}ms`);
      }
    }),
    ```

    Apply the same pattern to `lobby_state_sync`, `round_over`, `new_round_started`, `slot_snapped`, and `submission_feedback`. Use the event name as the log label.

    Keep each wrapper as a one-liner with `_t0` scoped inside the arrow function — do not mutate the surrounding scope or add refs for timing. The `_t0` prefix (underscore) avoids TS unused-variable warnings if the env guard prevents logging.

    After editing, run `npx tsc --noEmit` to verify no errors.

    NOTE: `performance.now()` is available in both browser and Node.js (Next.js server components). No import needed.

    CAUTION: Do NOT add timing to `useEffect`, `useRef`, or any render path — only to `onEvent` callbacks (called by the socket layer). This measures socket dispatch overhead, not React render time.
  </action>
