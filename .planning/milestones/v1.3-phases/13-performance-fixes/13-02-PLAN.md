Reduce initial JS bundle size by moving SentryUserSync (and its Supabase 645KB dependency) out of the main entry chunk via next/dynamic lazy loading.

SentryUserSync export shape (named export, not default):

```typescript
// src/components/SentryUserSync.tsx
export const SentryUserSync: React.FC = () => { ... }
```

next/dynamic API for named exports:
```typescript
const SentryUserSync = dynamic(
  () => import('@/components/SentryUserSync').then((m) => ({ default: m.SentryUserSync })),
  { ssr: false }
);
```

Why ssr: false is safe for SentryUserSync:
- SentryUserSync only calls supabase.auth.getUser() and sets Sentry user context
- No SSR value — it does not render any visible HTML
- ssr: false prevents it running during server rendering (correct — no SSR needed here)

Why Provider.tsx is the correct target (NOT layout.tsx):
- layout.tsx is a Server Component — Next.js docs: "When a Server Component dynamically
  imports a Client Component, automatic code splitting is currently not supported"
- Provider.tsx is a Client Component ("use client") — dynamic imports from Client Components
  produce genuine webpack code splits

Verification command from package.json:
- npm run analyze uses --webpack flag (Turbopack incompatible with @next/bundle-analyzer)
