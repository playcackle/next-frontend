<objective>
Fix LCP 4324ms (poor) by removing the router.refresh() call that fires during initial session restore.

Purpose: LCP is 3824ms after FCP — the server responds quickly but the hero section re-paints when useUser() calls router.refresh() on SIGNED_IN during initial auth state initialization. By gating router.refresh() to user-initiated events only (not the passive session restore), the hero section avoids a second paint cycle and LCP drops.

Output: src/hooks/useUser.ts updated so router.refresh() is not called when the INITIAL_SESSION event fires on page load.

Baseline: LCP 4324ms (from PERF-BASELINE.md, measured 2026-03-18)
Target: LCP below 4324ms — a measurable improvement. Good threshold is 2500ms.
</objective>

<task type="auto">
  <name>Task 2: Gate router.refresh() to user-initiated auth events only</name>
  <files>src/hooks/useUser.ts</files>
  <action>
    Modify the onAuthStateChange callback in src/hooks/useUser.ts to exclude INITIAL_SESSION
    from triggering router.refresh().

    Current code (lines 41-43):
    ```typescript
    if (event === 'SIGNED_IN' || event === 'SIGNED_OUT') {
      router.refresh();
    }
    ```

    The problem: Supabase fires SIGNED_IN on page load when restoring an existing session.
    The fix: add an early return for INITIAL_SESSION before the setUser/setLoading calls,
    so the auth state is set by loadUser() (the explicit getUser() call) rather than the
    listener, and router.refresh() is never called on passive session restore.

    Full implementation — add inside the onAuthStateChange callback, BEFORE setUser/setLoading:
    ```typescript
    supabase.auth.onAuthStateChange((event: AuthChangeEvent, session: Session | null) => {
      if (!isMounted) return;

      // INITIAL_SESSION fires on page load when restoring an existing session.
      // Do NOT call router.refresh() here — it causes a full Server Component re-fetch
      // and re-paints the LCP element, adding ~3800ms to the LCP timestamp.
      // Initial user state is set by the explicit loadUser() call above.
      if (event === 'INITIAL_SESSION') {
        return;
      }

      setUser(session?.user ?? null);
      setLoading(false);

      // Only refresh on explicit user-initiated events (login / logout)
      if (event === 'SIGNED_IN' || event === 'SIGNED_OUT') {
        router.refresh();
      }
    });
    ```

    No other changes. useRef import is not needed (no ref added). The loadUser() async call
    remains unchanged — it handles initial user state independently of the listener.

    Note: INITIAL_SESSION is a valid AuthChangeEvent string in @supabase/supabase-js v2+.
    The TypeScript union type includes it — no type assertion needed.
  </action>
  <verify>
    <automated>npx tsc --noEmit</automated>
  </verify>
  <done>TypeScript compiles without errors. useUser.ts returns early on INITIAL_SESSION without calling router.refresh(). SIGNED_IN and SIGNED_OUT still call router.refresh().</done>
</task>
