<objective>
Remove the Supabase dependency from UnifiedMessages by replacing useUser() with a lightweight Jotai atom.

Purpose: UnifiedMessages.tsx calls useUser() to get user.id for own-message styling. This pulls the full Supabase client into the gameroom hot-render path (re-renders every lobby_tick at 1Hz). While render cost is low, the Supabase SDK adds unnecessary surface area to the gameroom bundle and creates an auth subscription inside a component that has nothing to do with authentication.

The fix uses the existing Jotai state layer: add currentUserIdAtom to gameAtoms.ts, and replace useUser() in UnifiedMessages with useAtomValue(currentUserIdAtom). Find where in the gameroom the user.id is already known (room join flow) and set the atom there.

Output:
- gameAtoms.ts: new `currentUserIdAtom` writable atom (string | null)
- UnifiedMessages.tsx: useUser() removed, currentUserIdAtom used instead

Baseline: UnifiedMessages imports useUser() which transitively imports @supabase/ssr
Target: UnifiedMessages has zero Supabase imports
</objective>
